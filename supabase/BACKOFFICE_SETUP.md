# คู่มือเปิดใช้ระบบหลังบ้าน (Back-office)

ระบบหลังบ้านอยู่ที่หน้า **`/admin.html`** ของเว็บไซต์ (เช่น `https://siamelite.vercel.app/admin.html`)
เจ้าหน้าที่ใช้ **จัดการสินทรัพย์ · ดู Leads · เก็บเอกสาร** ได้จากหน้าเว็บเลย ไม่ต้องแตะโค้ด

มี **3 ขั้นตอนที่ต้องทำครั้งเดียว** ใน Supabase (ผมทำแทนไม่ได้เพราะไม่มีสิทธิ์เข้า dashboard)

---

## ขั้นที่ 1 — รัน SQL (สร้างตาราง + เปิดความปลอดภัย)

1. เข้า https://supabase.com/dashboard → เลือกโปรเจกต์ **loanbvamjfdlmyhstaby**
2. เมนูซ้าย → **SQL Editor** → **New query**
3. เปิดไฟล์ [`01_backoffice_schema.sql`](01_backoffice_schema.sql) → คัดลอกทั้งหมด → วางในช่อง → กด **Run**
4. ถ้าขึ้น "Success. No rows returned" = เรียบร้อย ✅

> ปลอดภัย: สคริปต์นี้ **ไม่ลบข้อมูลเดิม** แค่เพิ่มคอลัมน์ที่ขาด (`published`, `id`, ฯลฯ),
> ตั้งกฎความปลอดภัย (RLS) และสร้างที่เก็บไฟล์ 2 ถัง รันซ้ำได้ไม่มีปัญหา

---

## ขั้นที่ 2 — สร้างบัญชี login ให้เจ้าหน้าที่

1. เมนูซ้าย → **Authentication** → **Users** → ปุ่ม **Add user** → **Create new user**
2. ใส่ **อีเมล** + **รหัสผ่าน** ของเจ้าหน้าที่
3. ✅ ติ๊ก **Auto Confirm User** (เพื่อให้ login ได้ทันทีโดยไม่ต้องยืนยันอีเมล)
4. กด **Create user**
5. อยากได้หลายคน ก็เพิ่มได้เรื่อยๆ — ทุกคนที่อยู่ในนี้ login เข้าหลังบ้านได้

> 🔒 ใครที่ **ไม่มี** บัญชีในหน้านี้ จะเข้าหลังบ้านไม่ได้เลย แม้จะเปิด URL เจอก็ตาม

---

## ขั้นที่ 3 — เข้าใช้งาน

1. เปิด `https://siamelite.vercel.app/admin.html`
2. ใส่อีเมล + รหัสผ่านที่สร้างไว้ในขั้นที่ 2 → **เข้าสู่ระบบ**
3. เสร็จ! ใช้งานได้เลย

---

## ใช้งานยังไง

**แท็บ สินทรัพย์**
- กด **+ เพิ่มสินทรัพย์** เพื่อสร้างรายการใหม่ (กรอกครบ 3 ภาษา: ไทย/EN/中文)
- อัปโหลดรูปได้หลายรูป — **รูปแรก = รูปปก** ลากสลับลำดับได้
- ติ๊ก/เอาติ๊ก **เผยแพร่บนเว็บ** เพื่อซ่อน/แสดงรายการบนเว็บลูกค้า (บันทึกแล้วขึ้นเว็บทันที)
- แก้ไข/ลบได้จากปุ่มท้ายแต่ละแถว

**แท็บ Leads**
- ดูลูกค้าที่ติดต่อผ่านเว็บทั้งหมด กรองตามสถานะได้
- เปลี่ยนสถานะ (ใหม่ → ติดต่อแล้ว → ปิดการขาย/ไม่สำเร็จ) และเพิ่มโน้ตได้

**แท็บ เอกสาร**
- อัปโหลดสัญญา/โบรชัวร์/เอกสารวีซ่า เก็บไว้ในระบบอย่างปลอดภัย
- ผูกกับสินทรัพย์ได้ · ดาวน์โหลดได้ (ลิงก์มีอายุสั้นเพื่อความปลอดภัย)

---

## เกร็ดเทคนิค

- เว็บลูกค้าดึงสินทรัพย์จาก Supabase อัตโนมัติ ถ้า Supabase ล่มชั่วคราว จะ **สำรองด้วยไฟล์
  `assets/data/properties.json`** ที่มากับเว็บ (เว็บไม่มีวันขึ้นหน้าว่าง)
- รูปที่อัปโหลดใหม่จะเก็บใน Supabase Storage (ถัง `property-images`) เอกสารเก็บในถัง
  `documents` ซึ่งเข้าถึงได้เฉพาะเจ้าหน้าที่ที่ login แล้วเท่านั้น
- ความปลอดภัยบังคับที่ฝั่ง Supabase (Row Level Security) ไม่ใช่แค่ซ่อนหน้า —
  anon key ที่อยู่ในโค้ดหน้าเว็บทำได้แค่ "อ่านรายการที่เผยแพร่" กับ "รับฟอร์มลูกค้า" เท่านั้น
