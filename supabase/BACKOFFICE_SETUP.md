# คู่มือเปิดใช้ระบบหลังบ้าน (Back-office)

ระบบหลังบ้านอยู่ที่หน้า **`/admin.html`** (เช่น `https://siamelite.vercel.app/admin.html`)
ใช้ **จัดการสินทรัพย์ · ดู Leads · เก็บเอกสาร** ได้จากหน้าเว็บเลย ไม่ต้องแตะโค้ด

เปิดใช้แค่ **2 ครั้ง copy-paste** ใน Supabase แล้วก็ใช้ได้เลย (ผมทำแทนทั้งหมดไม่ได้เพราะไม่มีสิทธิ์เข้า dashboard ของคุณ)

---

## ขั้นที่ 1 — รัน SQL ตัวแรก (สร้างตาราง + ความปลอดภัย)

1. เข้า https://supabase.com/dashboard → เลือกโปรเจกต์ **loanbvamjfdlmyhstaby**
2. เมนูซ้าย → **SQL Editor** → **New query**
3. เปิดไฟล์ [`01_backoffice_schema.sql`](01_backoffice_schema.sql) → คัดลอกทั้งหมด → วาง → กด **Run**
4. ขึ้น "Success" = เรียบร้อย ✅ *(ไม่ลบข้อมูลเดิม 45 รายการ รันซ้ำได้)*

---

## ขั้นที่ 2 — สร้างแอดมิน 1 คน (รัน SQL ตัวที่สอง)

1. เปิดไฟล์ [`02_create_admin.sql`](02_create_admin.sql)
2. **แก้แค่ 2 บรรทัด** ที่มีเครื่องหมาย 👈 :
   - `admin_email` = อีเมลที่จะใช้ login (เช่น `admin@siamelite.co`)
   - `admin_password` = รหัสผ่านที่ตั้งเอง (อย่างน้อย 6 ตัว)
3. คัดลอกทั้งไฟล์ → วางใน SQL Editor → กด **Run**
4. ขึ้น "Admin user created" = เสร็จ ✅

> 🔑 รหัสผ่าน**ไม่ถูกเก็บใน git** — คุณพิมพ์ตอนรันเท่านั้น
> ลืมรหัส? แก้บรรทัด `admin_password` แล้วรัน `02` ซ้ำ = รีเซ็ตรหัสให้อัตโนมัติ

---

## ขั้นที่ 3 — เข้าใช้งาน

1. เปิด `https://siamelite.vercel.app/admin.html` (หรือกดลิงก์ **Staff** ที่ footer ล่างสุดของเว็บ)
2. ใส่อีเมล + รหัสผ่านจากขั้นที่ 2 → **เข้าสู่ระบบ**
3. เสร็จ! ใช้งานได้เลย

---

## ใช้งานยังไง

**แท็บ สินทรัพย์**
- กด **+ เพิ่มสินทรัพย์** เพื่อสร้างรายการใหม่ (กรอกครบ 3 ภาษา: ไทย/EN/中文)
- อัปโหลดรูปได้หลายรูป — **รูปแรก = รูปปก** ลากสลับลำดับได้
- ติ๊ก/เอาติ๊ก **เผยแพร่บนเว็บ** เพื่อซ่อน/แสดงรายการบนเว็บลูกค้า (บันทึกแล้วขึ้นเว็บทันที)
- แก้ไข/ลบได้จากปุ่มท้ายแต่ละแถว

**แท็บ Leads**
- ดูลูกค้าที่ติดต่อผ่านเว็บทั้งหมด กรองตามสถานะได้
- เปลี่ยนสถานะ (ใหม่ → ติดต่อแล้ว → ปิดการขาย/ไม่สำเร็จ) และเพิ่มโน้ตได้

**แท็บ เอกสาร**
- อัปโหลดสัญญา/โบรชัวร์/เอกสารวีซ่า เก็บไว้ในระบบอย่างปลอดภัย
- ผูกกับสินทรัพย์ได้ · ดาวน์โหลดได้ (ลิงก์มีอายุสั้นเพื่อความปลอดภัย)

---

## เกร็ดเทคนิค

- ใช้แอดมิน **คนเดียว** ก็พอ ถ้าอยากเพิ่มคนทีหลัง แค่แก้ `admin_email` ในไฟล์ `02` เป็นอีเมลใหม่แล้วรันอีกรอบ
- เว็บลูกค้าดึงสินทรัพย์จาก Supabase อัตโนมัติ ถ้า Supabase ล่มจะสำรองด้วยไฟล์ `assets/data/properties.json` (เว็บไม่มีวันขึ้นหน้าว่าง)
- ความปลอดภัยบังคับที่ฝั่ง Supabase (Row Level Security) — public key ในหน้าเว็บทำได้แค่ "อ่านรายการที่เผยแพร่" กับ "รับฟอร์มลูกค้า" เท่านั้น การแก้ไขทุกอย่างต้อง login ก่อน
